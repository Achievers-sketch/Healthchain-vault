# **App Name**: HealthChain Vault

## Core Features:

- Encrypted Data Vault: AES-256 client-side encryption of health data with user private key for secure storage.
- Decentralized Storage: Secure storage of encrypted health data on IPFS. Support metadata storage schema.
- DID Authentication: DID-based identity authentication for secure user onboarding.
- Smart Contract Consent: Smart contracts for consent approval, access revocation, and read-access logging.
- Selective Sharing with ZK-Proofs: Enable selective sharing of health data using zero-knowledge proofs, allowing users to prove certain attributes without revealing the data itself. Reasoning tool embedded in LLM that conditionally shares parts of personal information.
- Wearable Integration: Support integration with wearable devices like Apple Health and Fitbit for automated data capture.
- Emergency Access: Provide emergency access via QR code for critical situations.

## Style Guidelines:

- Primary color: Forest green (#388E3C), reflecting health, nature, and security.
- Background color: Very light green (#F1F8E9) to create a clean, trustworthy feel.
- Accent color: Teal (#008080) to complement the primary green and indicate important interactive elements.
- Body font: 'Inter' (sans-serif) for clean and readable text.
- Headline font: 'Space Grotesk' (sans-serif), providing a futuristic and tech-oriented feel.
- Use a set of simple, consistent line icons representing health, security, and data-related concepts.
- Maintain a clean, intuitive layout with clear visual hierarchy. Prioritize accessibility and ease of navigation.
